## 1. Đáp án lựa chọn

**Chọn phương án B.**

Phương án B là prompt tốt nhất vì mô tả rõ vai trò, công nghệ test cần dùng, phạm vi kiểm thử, các trường hợp biên và định dạng đầu ra mong muốn.

## 2. Phân tích vì sao phương án B tối ưu nhất

Phương án B:

```text
Hãy đóng vai trò là một chuyên gia kiểm thử phần mềm (Senior QA Engineer).
Nhiệm vụ của bạn là viết một class unit test sử dụng JUnit 5 và Assertions để kiểm thử toàn diện class PasswordValidator.
...
Sử dụng Parameterized Test (@ParameterizedTest, @ValueSource)...
Phải bao phủ toàn bộ các trường hợp biên...
```

### Lý do tối ưu

**1. Có vai trò rõ ràng**

Prompt yêu cầu AI đóng vai **Senior QA Engineer**, giúp AI tập trung vào tư duy kiểm thử, bao phủ case biên và kiểm thử logic thay vì chỉ viết code đơn giản.

**2. Chỉ rõ công nghệ kiểm thử**

Prompt yêu cầu dùng:

```java
JUnit 5
Assertions
@ParameterizedTest
@ValueSource
```

Điều này giúp AI sinh đúng loại test class cần thiết, thay vì dùng JUnit 4 hoặc test thủ công từng case.

**3. Bao phủ đầy đủ Edge Cases**

Prompt liệt kê rõ các trường hợp cần test:

- Mật khẩu `null`.
- Chuỗi rỗng.
- Mật khẩu chứa khoảng trắng.
- Mật khẩu quá ngắn: 7 ký tự.
- Mật khẩu quá dài: 21 ký tự.
- Thiếu chữ hoa.
- Thiếu chữ thường.
- Thiếu chữ số.
- Thiếu ký tự đặc biệt.
- Mật khẩu hợp lệ ở biên 8 ký tự.
- Mật khẩu hợp lệ ở biên 20 ký tự.

Đây là điểm rất quan trọng vì kiểm thử password validator cần kiểm tra cả luồng đúng và luồng sai.

**4. Có yêu cầu định dạng đầu ra rõ ràng**

Prompt yêu cầu trả về mã nguồn Java JUnit 5 hoàn chỉnh và giải thích bằng tiếng Việt. Nhờ vậy, kết quả dễ nộp bài và dễ kiểm chứng.

## 3. Phân tích lý do loại trừ các phương án còn lại

### Phương án A

```text
Hãy viết unit test cho class PasswordValidator của tôi, test các trường hợp mật khẩu hợp lệ và không hợp lệ.
```

Phương án A quá chung chung.

Nhược điểm:

- Không yêu cầu dùng JUnit 5.
- Không yêu cầu dùng Parameterized Test.
- Không liệt kê các trường hợp biên.
- Không nhắc đến `null`, chuỗi rỗng, quá ngắn, quá dài.
- Không yêu cầu test thiếu từng thành phần bắt buộc.
- AI dễ sinh test thiếu case hoặc chỉ viết vài test cơ bản.

Ví dụ AI có thể chỉ sinh 2 test:

```java
validPassword_shouldReturnTrue()
invalidPassword_shouldReturnFalse()
```

Như vậy không đủ bao phủ logic.

---

### Phương án C

```text
Hãy viết lại hàm PasswordValidator sử dụng regex ngắn hơn để tôi không phải viết unit test.
```

Phương án C sai mục tiêu đề bài.

Nhược điểm:

- Đề bài yêu cầu viết unit test, không yêu cầu viết lại hàm.
- Việc dùng regex ngắn hơn không đảm bảo logic đúng.
- Không có test thì không kiểm chứng được các trường hợp biên.
- Có thể làm thay đổi logic ban đầu của `PasswordValidator`.
- Đi ngược lại mục tiêu kiểm thử chất lượng phần mềm.

Vì vậy, phương án C không phù hợp.

## 4. Mã nguồn JUnit 5 test class

Giả sử class `PasswordValidator` có method:

```java
public boolean isValid(String password)
```

Test class hoàn chỉnh:

```java
import org.junit.jupiter.api.DisplayName;
import org.junit.jupiter.api.Test;
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.NullSource;
import org.junit.jupiter.params.provider.ValueSource;

import static org.junit.jupiter.api.Assertions.assertFalse;
import static org.junit.jupiter.api.Assertions.assertTrue;

class PasswordValidatorTest {

    private final PasswordValidator passwordValidator = new PasswordValidator();

    @ParameterizedTest
    @ValueSource(strings = {
            "Aa1@aaaa",
            "Abcdef1@",
            "A1@bcdef",
            "StrongPass1@",
            "Abcdefghij12345@XYZ"
    })
    @DisplayName("Should return true for valid passwords")
    void shouldReturnTrueForValidPasswords(String password) {
        assertTrue(passwordValidator.isValid(password));
    }

    @ParameterizedTest
    @ValueSource(strings = {
            "Aa1@aaa"
    })
    @DisplayName("Should return false when password is shorter than 8 characters")
    void shouldReturnFalseWhenPasswordIsTooShort(String password) {
        assertFalse(passwordValidator.isValid(password));
    }

    @ParameterizedTest
    @ValueSource(strings = {
            "Aa1@aaaaaaaaaaaaaaaaa"
    })
    @DisplayName("Should return false when password is longer than 20 characters")
    void shouldReturnFalseWhenPasswordIsTooLong(String password) {
        assertFalse(passwordValidator.isValid(password));
    }

    @ParameterizedTest
    @ValueSource(strings = {
            "",
            "   "
    })
    @DisplayName("Should return false when password is empty or blank")
    void shouldReturnFalseWhenPasswordIsEmptyOrBlank(String password) {
        assertFalse(passwordValidator.isValid(password));
    }

    @ParameterizedTest
    @NullSource
    @DisplayName("Should return false when password is null")
    void shouldReturnFalseWhenPasswordIsNull(String password) {
        assertFalse(passwordValidator.isValid(password));
    }

    @ParameterizedTest
    @ValueSource(strings = {
            "aa1@aaaa",
            "password1@"
    })
    @DisplayName("Should return false when password does not contain uppercase letter")
    void shouldReturnFalseWhenMissingUppercaseLetter(String password) {
        assertFalse(passwordValidator.isValid(password));
    }

    @ParameterizedTest
    @ValueSource(strings = {
            "AA1@AAAA",
            "PASSWORD1@"
    })
    @DisplayName("Should return false when password does not contain lowercase letter")
    void shouldReturnFalseWhenMissingLowercaseLetter(String password) {
        assertFalse(passwordValidator.isValid(password));
    }

    @ParameterizedTest
    @ValueSource(strings = {
            "Aa@aaaaa",
            "Password@"
    })
    @DisplayName("Should return false when password does not contain digit")
    void shouldReturnFalseWhenMissingDigit(String password) {
        assertFalse(passwordValidator.isValid(password));
    }

    @ParameterizedTest
    @ValueSource(strings = {
            "Aa1aaaaa",
            "Password1"
    })
    @DisplayName("Should return false when password does not contain special character")
    void shouldReturnFalseWhenMissingSpecialCharacter(String password) {
        assertFalse(passwordValidator.isValid(password));
    }

    @ParameterizedTest
    @ValueSource(strings = {
            "Aa1@ aaa",
            "Aa1@aaaa ",
            " Aa1@aaaa"
    })
    @DisplayName("Should return false when password contains whitespace")
    void shouldReturnFalseWhenPasswordContainsWhitespace(String password) {
        assertFalse(passwordValidator.isValid(password));
    }

    @Test
    @DisplayName("Should return true when password length is exactly 8 characters")
    void shouldReturnTrueWhenPasswordLengthIsExactly8() {
        String password = "Aa1@aaaa";

        assertTrue(passwordValidator.isValid(password));
    }

    @Test
    @DisplayName("Should return true when password length is exactly 20 characters")
    void shouldReturnTrueWhenPasswordLengthIsExactly20() {
        String password = "Aa1@abcdefghijklmnop";

        assertTrue(passwordValidator.isValid(password));
    }

    @Test
    @DisplayName("Should return false when password length is exactly 7 characters")
    void shouldReturnFalseWhenPasswordLengthIsExactly7() {
        String password = "Aa1@aaa";

        assertFalse(passwordValidator.isValid(password));
    }

    @Test
    @DisplayName("Should return false when password length is exactly 21 characters")
    void shouldReturnFalseWhenPasswordLengthIsExactly21() {
        String password = "Aa1@abcdefghijklmnopq";

        assertFalse(passwordValidator.isValid(password));
    }
}
```

## 5. Giải thích bộ test

Bộ test trên kiểm tra đầy đủ các nhóm case quan trọng:

- **Valid case**: mật khẩu đúng đủ điều kiện.
- **Boundary case**: độ dài đúng 8, đúng 20, sai ở 7 và 21 ký tự.
- **Null/empty case**: kiểm tra `null`, `""`, `"   "`.
- **Missing rule case**:
  - Thiếu chữ hoa.
  - Thiếu chữ thường.
  - Thiếu chữ số.
  - Thiếu ký tự đặc biệt.

- **Whitespace case**: kiểm tra mật khẩu có khoảng trắng.

Việc dùng `@ParameterizedTest` giúp gom nhiều input cùng loại vào một test, giảm lặp code và tăng độ rõ ràng.
